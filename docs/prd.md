## 1. 제품 개요

### 1.1 제품 비전
한국투자증권 Open API를 활용하여 KOSPI/KOSDAQ 시장의 관심종목/보유종목에 대해 검증된 퀀트 전략 기반으로 자동 매매를 수행하는 웹 애플리케이션.

### 1.2 목표 사용자
- **MVP:** 개인 사용자 1인 (개발자 본인)
- **향후:** 일반 개인 투자자에게 오픈 (멀티 유저 지원)

### 1.3 핵심 가치
- 감정 배제: 규칙 기반 매매로 감정적 판단 제거
- 검증된 전략: 오픈소스 기반의 퀀트 전략 활용
- 24/7 모니터링: 장중 자동 감시 및 주문 실행
- 투명성: 모든 매매 판단 근거와 히스토리 기록

## 2. 핵심 기능 요구사항

### 2.1 한국투자증권 API 연동

#### 2.1.1 인증 및 계정 관리
- OAuth 2.0 기반 인증 (appkey, appsecret → access_token 발급)
- 토큰 자동 갱신 (만료 전 5분 갱신, 24시간 유효)
- 모의투자/실전투자 환경 전환 지원
- **MVP에서는 모의투자 계정으로 먼저 테스트 후 실전 전환**

#### 2.1.2 사용할 API 목록
| API 카테고리 | 용도 | 방식 |
|---|---|---|
| 현재가 조회 | 종목 실시간 시세 | REST |
| 일봉/주봉/월봉 조회 | 기술적 분석용 과거 데이터 | REST |
| 분봉 데이터 조회 | 당일 상세 시세 | REST |
| 잔고 조회 | 보유종목 및 예수금 확인 | REST |
| 매수/매도 주문 | 자동 주문 실행 | REST |
| 주문 정정/취소 | 미체결 주문 관리 | REST |
| 실시간 체결 | 체결 알림 | WebSocket |
| 실시간 호가 | 호가 모니터링 | WebSocket |

#### 2.1.3 추천 라이브러리
- **python-kis (PyKIS)**: 한국투자증권 커뮤니티 라이브러리 (GitHub: Soju06/python-kis)
  - 국내/해외 통합 인터페이스
  - WebSocket 자동 재연결
  - Thread-safe 토큰 관리
  - Typing 완전 지원
- **대안:** 한국투자증권 공식 샘플 코드 (GitHub: koreainvestment/open-trading-api)

### 2.2 관심종목/보유종목 관리

#### 2.2.1 관심종목 (Watchlist)
- 종목코드 또는 종목명으로 검색 및 추가
- 관심종목 그룹 분류 (예: "성장주", "배당주", "단기 트레이딩")
- 종목별 전략 할당 (어떤 전략으로 매매할지 설정)
- 목표 매수가 / 매수 조건 설정

#### 2.2.2 보유종목 (Portfolio)
- 한국투자증권 API 연동으로 실시간 보유종목 동기화
- 종목별 매입가, 수량, 수익률, 평가금액 표시
- 종목별 매도 전략 / 손절·익절 라인 설정

### 2.3 자동매매 전략 엔진

#### 2.3.1 전략 방향성
**복합 전략 (기술적 분석 + 가치 분석 + 모멘텀)** 을 지원하며, 사용자가 전략을 선택·조합할 수 있도록 함.

#### 2.3.2 기술적 분석 지표 (Technical Indicators)
오픈소스 라이브러리 **TA-Lib** 또는 **pandas-ta** 활용

| 지표 | 설명 | 활용 |
|---|---|---|
| SMA / EMA | 단순/지수 이동평균 | 골든크로스/데드크로스 |
| RSI (14일) | 상대강도지수 | 과매수(>70)/과매도(<30) |
| MACD | 이동평균 수렴확산 | 시그널 크로스 |
| Bollinger Bands | 볼린저 밴드 | 밴드 이탈/회귀 |
| ATR | 평균진정범위 | 변동성 기반 손절폭 설정 |
| Volume Ratio | 거래량 비율 | 거래량 급증 감지 |

#### 2.3.3 가치 분석 지표 (Fundamental Indicators)
한국투자증권 API 또는 외부 데이터 소스에서 수급

| 지표 | 설명 | 활용 |
|---|---|---|
| PER | 주가수익비율 | 저평가 스크리닝 |
| PBR | 주가순자산비율 | 자산 대비 저평가 |
| ROE | 자기자본이익률 | 수익성 판단 |
| 배당수익률 | 연간 배당금 / 주가 | 배당 투자 기준 |

#### 2.3.4 모멘텀/추세 전략
| 전략 | 설명 | 매매 신호 |
|---|---|---|
| 듀얼 모멘텀 | 상대모멘텀 + 절대모멘텀 | 최근 N개월 수익률 기준 |
| 추세 추종 | 이동평균 기울기 기반 | 상승 추세 진입, 하락 추세 탈출 |
| 변동성 돌파 | 래리 윌리엄스 전략 변형 | 전일 변동폭의 K% 돌파 시 매수 |

#### 2.3.5 MVP 기본 전략 (프리셋)
사용자가 바로 사용 가능한 검증된 전략 프리셋 3종:

**전략 1: 골든크로스 + RSI 복합**
```
매수 조건: SMA(20) > SMA(60) AND RSI(14) < 40 AND 거래량 > 20일 평균의 1.5배
매도 조건: SMA(20) < SMA(60) OR RSI(14) > 75
손절: 매입가 대비 -5%
익절: 매입가 대비 +15%
```

**전략 2: 가치 + 모멘텀 하이브리드**
```
매수 조건: PER < 업종 평균의 0.7배 AND PBR < 1.0 AND ROE > 10% AND 3개월 수익률 > 0
매도 조건: PER > 업종 평균의 1.2배 OR 3개월 수익률 < -10%
리밸런싱: 월 1회
```

**전략 3: 볼린저 밴드 반전**
```
매수 조건: 종가 < BB_Lower AND RSI(14) < 30
매도 조건: 종가 > BB_Upper OR 종가 > BB_Middle (보수적)
손절: 매입가 대비 -7%
```

#### 2.3.6 전략 커스터마이징
- 위 기본 전략의 파라미터 조정 가능 (기간, 배수, 비율 등)
- 여러 지표를 AND/OR로 조합하는 커스텀 전략 빌더
- 전략별 투자 비중 설정 (예: 전략 1에 50%, 전략 2에 30%, 전략 3에 20%)

### 2.4 손절/익절 자동 설정

#### 2.4.1 손절 (Stop Loss)
| 방식 | 설명 |
|---|---|
| 고정 비율 손절 | 매입가 대비 -N% 도달 시 매도 |
| 트레일링 스톱 | 고점 대비 -N% 하락 시 매도 (수익 구간 보호) |
| ATR 기반 손절 | ATR의 N배 하락 시 매도 (변동성 반영) |

#### 2.4.2 익절 (Take Profit)
| 방식 | 설명 |
|---|---|
| 고정 비율 익절 | 매입가 대비 +N% 도달 시 매도 |
| 분할 익절 | 목표가 도달 시 50% 매도, 추가 도달 시 나머지 매도 |
| 트레일링 익절 | 목표 수익률 도달 후 트레일링 스톱으로 전환 |

### 2.5 백테스팅 엔진

#### 2.5.1 오픈소스 프레임워크 선택
**권장: VectorBT** (Python 벡터화 백테스팅 프레임워크)
- pandas/numpy 네이티브 — KIS API DataFrame을 그대로 입력 가능
- 벡터화 연산으로 Backtrader 대비 수십~수백 배 빠른 속도
- 파라미터 조합 수천 개를 동시에 테스트 (전략 최적화에 탁월)
- 성과 지표 자동 계산 (Sharpe, MDD, CAGR 등)
- Apache 2.0 라이선스 (상용 배포 가능)
- 2025년 기준 활발히 유지보수 중

**대안:**
- **Zipline-reloaded**: Quantopian 후속, 이벤트 드리븐 방식, 거래 달력 지원
- **Backtesting.py** (kernc/backtesting.py): 직관적 API, 프로토타이핑에 적합 (AGPL-3.0)
- **bt** (pmorissette/bt): 포트폴리오 수준 백테스팅, MIT 라이선스

#### 2.5.2 백테스팅 기능 명세
- 한국투자증권 API에서 일봉/주봉 데이터를 가져와 VectorBT에 연동
- 최소 3년 ~ 최대 10년 과거 데이터 지원
- 성과 지표 출력:
  - 총 수익률, 연평균 수익률 (CAGR)
  - 최대 낙폭 (MDD)
  - 샤프 비율 (Sharpe Ratio)
  - 승률 (Win Rate)
  - 손익비 (Profit Factor)
- 벤치마크 비교 (KOSPI 지수 대비)
- 결과 차트 시각화 (수익 곡선, 드로우다운)

## 3. 시스템 아키텍처

### 3.1 기술 스택 (권장)

#### Frontend
- **Next.js 16+ (App Router)** + TypeScript
  - 선택 이유: SSR/SSG 지원, API Routes로 간단한 BFF 구현, React 생태계 활용
  - 빠른 프로토타이핑과 추후 확장성 모두 확보
- **TailwindCSS** + **shadcn/ui**: 빠른 UI 개발
- **Recharts** 또는 **Lightweight Charts (TradingView)**: 차트 시각화
- **TanStack Query (React Query)**: 서버 상태 관리

#### Backend
- **FastAPI (Python)**
  - 선택 이유: 한국투자증권 라이브러리(python-kis)가 Python 기반, 퀀트 라이브러리(TA-Lib, VectorBT, pandas) 모두 Python
  - 비동기 지원, 자동 API 문서 생성 (Swagger)
  - WebSocket 지원
- **Celery + Redis**: 백그라운드 작업 스케줄링 (전략 실행, 정기 분석)
- **APScheduler** (경량 대안): MVP에서 간단한 스케줄링용

#### Database
- **PostgreSQL**: 메인 데이터베이스 (거래 내역, 전략 설정, 사용자 데이터)
- **Redis**: 캐싱, 실시간 시세 버퍼, 세션 관리, 작업 큐

#### Infra (MVP)
- **Docker Compose**: 로컬 개발 및 단일 서버 배포
- **향후:** AWS/GCP + Kubernetes

## 6. 안전장치 및 리스크 관리

### 6.1 매매 안전장치 (MVP 필수)
- **일일 최대 손실 한도:** 설정 금액 초과 시 당일 매매 자동 중단
- **일일 최대 주문 횟수:** 과잉 매매 방지
- **단일 종목 최대 투자 비중:** 포트폴리오의 N% 초과 방지
- **주문 전 확인:** 자동매매 ON/OFF 마스터 스위치
- **긴급 전체 매도:** 원클릭 전 종목 시장가 매도

### 6.2 시스템 안전장치
- API 장애 시 재시도 로직 (최대 3회, 지수 백오프)
- WebSocket 연결 끊김 시 자동 재연결
- 중복 주문 방지 (동일 종목 N초 이내 중복 주문 차단)
- 모든 주문/판단 로그 영구 저장
- 시스템 에러 발생 시 텔레그램 즉시 알림

### 6.3 데이터 보안
- API 키 암호화 저장 (환경변수 또는 Vault)
- HTTPS 전용 통신
- JWT 기반 인증 (멀티유저 확장 시)

## 8. 성공 지표 (KPI)

### MVP 기준
- 모의투자 환경에서 전략 기반 자동 매매가 정상 작동하는가
- 백테스팅 결과와 실제 매매 결과 간 괴리가 합리적인 범위인가
- 텔레그램 알림이 실시간으로 도착하는가
- 시스템 에러 없이 장중 연속 운영이 가능한가 (최소 5일 연속)

### 향후 지표
- 전략별 실전 수익률 vs. 벤치마크(KOSPI)
- 시스템 가동률 99% 이상
- 주문 실행 지연 시간 < 3초

## 9. 제약사항 및 고려사항

### 9.1 한국투자증권 API 제약
- REST API 초당 호출 제한 (실전: 초당 20건, 모의: 초당 5건)
- WebSocket 동시 등록 종목 수 제한 (세션당 41건)
- 액세스 토큰 하루 발급 횟수 제한
- 장중 시간만 주문 가능 (09:00~15:30)
- 3개월 이상 미거래 시 API 서비스 자동 해지

### 9.2 법적/규제 고려
- 자동매매 시스템은 투자 조언에 해당하지 않음 (면책 조항 필요)
- 멀티유저 오픈 시 금융투자업 규제 검토 필요
- 사용자 데이터(API 키 등) 보호 의무

### 9.3 기술적 제약
- TA-Lib C 라이브러리 설치 필요 (Docker 환경 권장)
- VectorBT Apache 2.0 라이선스 → 상용 배포 시 라이선스 문제 없음
- 한국 주식 시장 데이터의 과거 데이터 접근성 한계

## 부록 A: 한국투자증권 API 셋업 가이드

### A.1 사전 준비
1. 한국투자증권 계좌 개설 (비대면 가능)
2. HTS ID 등록
3. KIS Developers 서비스 신청 (https://apiportal.koreainvestment.com)
4. 앱키(App Key) / 시크릿키(App Secret) 발급
5. 모의투자 계정 별도 신청

### A.2 개발 환경
- API 호출 도메인:
  - 실전: `https://openapi.koreainvestment.com:9443`
  - 모의: `https://openapivts.koreainvestment.com:29443`
- 인증: OAuth 2.0 (appkey + appsecret → access_token)
- 데이터 형식: JSON

### A.3 주의사항
- appkey/appsecret은 절대 노출 금지 (Git 커밋 주의)
- TLS 1.2 이상만 지원 (2025.12.12 이후 TLS 1.0/1.1 미지원)
- API Rate Limit 준수 필수
